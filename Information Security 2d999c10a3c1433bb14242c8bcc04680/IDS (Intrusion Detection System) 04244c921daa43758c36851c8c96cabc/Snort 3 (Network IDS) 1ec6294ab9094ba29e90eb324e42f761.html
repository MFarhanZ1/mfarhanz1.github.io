Snort 3 (Network IDS)

Snort adalah free dan open source network intrusion prevention system (NIPS) dan network intrusion detection system (NIDS), yang di kembangkan oleh Martin Roesch tahun 1998. Snort saat ini di kembangkan oleh Sourcefire, dimana Roesch adalah founder dan CTO. Tahun 2009, Snort terpilih sebagai ‘greatest open source software of all time.

Snort’s open source network-based intrusion detection system (NIDS) yang mempunyai kemampuan untuk melakukan analisa traffic secara real time dan packet logging dari jaringan Internet Protocol (IP). Snort melakukan analisa protocol, pencarian content, pencocokan content. Snort juga dapat digunakan untuk mendeteksi serangan, termasuk, tapi tidak terbatas pada, operating system fingerprinting attempts, common gateway interface, buffer overflows, server message block probes, dan stealth port scans.

Snort dapat dikonfigurasi menggunakan tiga mode utama: sniffer, packet logger, dan network intrusion detection.

  • Mode sniffer - snort akan membaca paket yang lewat dan menampilkan ke layar.
  • Mode logger - snort akan mencatat paket yang lewat ke disk.
  • Mode Intrusion Detection - snort akan memonitor semua traffic yang lewat, membandingkan dengan rule yang di definisikan oleh user.

Ada beberapa tool yang bisa di kawinkan dengan snort untuk administrasi, reporting, dan analisa log, seperti,

  • Snorby [1]
  • BASE [2]
  • RazorBack [3]

1.1 Sedikit Mengganti Konfigurasi snort.lua (FIX Ping Failed Detected)

Jika kita mengikuti tutorial pada link diatas sebelumnya akan menyebabkan sedikit masalah, yakni setelah memasang update rules (knowledge) deteksi ICMP Ping Scan malah menjadi gagal terdeteksi, hal tersebut terjadi dikarenakan local.rules ICMP bentrok sama community rules nya.

Solusi Buat Solvenya ada 2 Solusi (Pilih Salah Satu):

  • Kalau mau solusi permanent,
    # edit snort.lua files
sudo nano /usr/local/etc/snort/snort.lua

    lalu kasih comment (--) di include = rule_path… "/rules community" lihat gambar dibawah:

    # lalu tinggal reload deh service snort3 yang udah di register sebelumnya gunain
sudo service snort3 restart
  • Kalau mau solusi sementara, ganti command tiap mau jalanin snort3
    # matikan dulu service snort3 nya
sudo systemctl stop snort3

# baru kita jalankan snort3 manual dengan copykan command ini di terminal
sudo snort -c /usr/local/etc/snort/snort.lua -R /usr/local/etc/rules/local.rules -i wlo1 -A alert_fast -s 65535 -k none

' Notes: Apasih arti -R itu?
	1. Nah jadi simpelnya itu tuh menspesifik file rules yang akan dibaca snort
	2. Maka, jika kalian tidak menambahkan -R pada command, snort akan secara otomatis
		 membaca rules yang udah di tetapin di config snort.lua 
     yakni rules yang terletak pada bagian: ips include = rule path....'

Monitoring Log Snort3 Secara Live

Kalian bisa memonitoring alert dari paket asing sesuai dengan rules yang diterapkan sebelumnya

sudo tail -f /var/log/snort/alert_fast.txt

# atau versi UI lebih interaktif menarik bisa gunain perintah
sudo lnav /var/log/snort/alert_fast.txt

semua status log yang sedang terjadi di service snort akan keluar secara live di terminal anda, anda bisa memonitor status ip yang sedang di banned oleh fail2ban dan sebaliknya.
