Web Server

Apache HTTP Server, atau yang lebih dikenal sebagai Apache, adalah perangkat lunak server web open-source yang sangat populer. Pada sistem operasi Ubuntu, Apache sering kali digunakan sebagai web server utama karena kemudahan penggunaannya dan kemampuan yang solid dalam menangani permintaan web.

Berikut adalah beberapa poin tentang Apache HTTP Server pada Ubuntu:

  1. Server Web: Apache berperan sebagai server web yang mengelola permintaan HTTP dari klien (seperti browser web) dan memberikan konten yang diminta, seperti halaman web, gambar, file CSS, JavaScript, dan berbagai konten web lainnya.
  1. Open Source: Apache adalah perangkat lunak open-source, artinya kode sumbernya dapat diakses, dimodifikasi, dan didistribusikan kembali sesuai dengan lisensi yang berlaku.
  1. Konfigurasi: Apache memiliki file konfigurasi utama bernama httpd.conf yang berada di dalam direktori /etc/apache2 (lokasi default di Ubuntu). Di sini, administrator dapat mengatur berbagai aspek server web, termasuk port yang digunakan, direktori default untuk file web, modul yang diaktifkan, dan banyak lagi.
  1. Modularitas: Apache memungkinkan penggunaan modul tambahan yang dapat diaktifkan atau dinonaktifkan sesuai kebutuhan. Modul ini memperluas fungsionalitas Apache, seperti pengompresi data, keamanan, integrasi dengan bahasa pemrograman tertentu (seperti PHP), dan banyak lagi.
  1. Performa dan Stabilitas: Apache terkenal akan kehandalan, kestabilan, dan kinerjanya yang baik dalam menangani lalu lintas web yang besar. Dengan konfigurasi yang tepat, Apache dapat menangani banyak permintaan secara bersamaan dengan efisien.
  1. Dukungan Komunitas yang Luas: Karena popularitasnya yang besar, terdapat sumber daya yang kaya dalam bentuk dokumentasi, forum, dan komunitas yang dapat membantu dalam mengatasi masalah dan memberikan saran terkait penggunaan Apache.

Dengan menggunakan Apache sebagai web server pada sistem Ubuntu, pengguna dapat menyajikan konten web dengan aman dan dapat diandalkan, serta memiliki kontrol yang kuat terhadap konfigurasi dan penyesuaian berbagai fitur server web sesuai kebutuhan mereka.

Instlasi Web Server (Apache Web Server)

Untuk menginstal Apache Web Server di Ubuntu, Anda dapat mengikuti langkah-langkah berikut menggunakan terminal:

  1. Update Paket yang Tersedia: Sebelum menginstal Apache, pastikan paket yang tersedia dalam sistem Ubuntu sudah diperbarui. Jalankan perintah berikut:
    sudo apt update
sudo apt upgrade
  1. Instal Apache: Setelah memperbarui paket, instal Apache dengan menjalankan perintah:
    sudo apt install apache2
  1. Verifikasi Status Apache: Setelah proses instalasi selesai, cek status Apache untuk memastikan bahwa layanan telah berhasil diinstal dengan baik. Gunakan perintah:
    sudo systemctl status apache2

    Jika layanan Apache berjalan dengan baik, Anda akan melihat pesan yang menyatakan bahwa Apache sedang berjalan.

  1. Buat Web Directory Root & Add Nama Domain Baru

  1. Uji Coba Akses Web Server: Buka browser web dan masukkan alamat IP server Ubuntu atau localhost atau nama domain yang udah kita buat yakni hanz-ssl.com (jika Anda menginstal Apache di komputer lokal) ke dalam bar alamat. Jika Apache telah berhasil diinstal, Anda akan melihat halaman selamat datang Apache, yang menunjukkan bahwa server web sedang berjalan.
  1. Konfigurasi Firewall (Opsional): Jika Anda menggunakan firewall pada server, pastikan port 80 (port default untuk HTTP) dibuka agar pengguna dapat mengakses web server. Gunakan perintah:
    sudo ufw allow 'Apache'
sudo ufw allow 'Apache Full'

Dengan mengikuti langkah-langkah ini, Anda akan berhasil menginstal dan menjalankan Apache Web Server di sistem Ubuntu Anda. Pastikan untuk memeriksa dokumentasi resmi Apache untuk konfigurasi lebih lanjut dan pengaturan tambahan sesuai dengan kebutuhan Anda.

Setting & Config Apache SSL Self Signed Certified Authority

Step 1 - Aktifkan SSL Module: Aktifkan SSL & restart apache

sudo a2enmod ssl
sudo service apache2 restart

Step 2 - buat self-sign SSL certificate: Buat folder SSL & buat self-signed certificate

sudo mkdir /etc/apache2/ssl
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/apache2/ssl/apache.key -out /etc/apache2/ssl/apache.crt

Artinya

  • openssl - perintah untuk operasi SSL
  • req - X.509 certificate signing request (CSR)
  • X.509 - self-signed tanpa certified auth (biar gratis)
  • nodes - berarti key file tidak perlu di amankan dengan passphrase
  • days 365 - durasi key tersebut valid, dimana dalam contoh ini yakni 1 tahun
  • newkey rsa:2048: buat CSR dan private key bersamaan. RSA key panjangnya 2048 bit.
  • keyout: nama output file dari private key yang dibuat.
  • out: nama output file certificate digital yang dibuat.

Setelah tekan "ENTER", ada beberapa pertanyaan atau jawaban yang perlu dilakukan dalam proses, diantaranya yakni,

Country Name (2 letter code) [AU]:ID
State or Province Name (full name) [Some-State]:Riau
Locality Name (eg, city) []:Pekanbaru
Organization Name (eg, company) [Internet Widgits Pty Ltd]:Hanz Cert Auth Ltd
Organizational Unit Name (eg, section) []:Hanz Dept. of CA Ultimate Service
Common Name (e.g. server FQDN or YOUR name) []:hanz-ssl.com
Email Address []:support@hanz-ssl.com

Sertificate & key akan di letakan di directory /etc/apache2/ssl

Step 3 - Konfigurasi Apache untuk menggunakan SSL

# kita akan mengcopy kan dulu file konfig default bawaan dari apache ssl
sudo cp -r /etc/apache2/sites-available/default-ssl.conf /etc/apache2/sites-available/hanz-ssl.conf

# lalu kita perlu memperbarui konfigurasinya, edit dengan nano text editor
sudo nano /etc/apache2/sites-available/hanz-ssl.conf

Tampilan file sebelum di edit adalah sebagai berikut,

<IfModule mod_ssl.c>
    <VirtualHost _default_:443>
        ServerAdmin webmaster@localhost
        DocumentRoot /var/www/html
        ErrorLog ${APACHE_LOG_DIR}/error.log
        CustomLog ${APACHE_LOG_DIR}/access.log combined
        SSLEngine on
        SSLCertificateFile /etc/ssl/certs/ssl-cert-snakeoil.pem
        SSLCertificateKeyFile /etc/ssl/private/ssl-cert-snakeoil.key
        <FilesMatch "\.(cgi|shtml|phtml|php)$">
                        SSLOptions +StdEnvVars
        </FilesMatch>
        <Directory /usr/lib/cgi-bin>
                        SSLOptions +StdEnvVars
        </Directory>
        BrowserMatch "MSIE [2-6]" \
                        nokeepalive ssl-unclean-shutdown \
                        downgrade-1.0 force-response-1.0
        BrowserMatch "MSIE [17-9]" ssl-unclean-shutdown
    </VirtualHost>
</IfModule>

Hal yang perlu kita ubah di file tersebut, yakni:

  • ServerAdmin (boleh di ubah, boleh juga engga kok)
  • ServerName (belum ada di file default, jadi kita tambahkan manual)
  • ServerAlias (belum ada di file default, jadi kita tambahkan manual)
  • DocumentRoot (ini *wajib, samakan dengan folder yang dah kita buat di tahap sebelumnya)
