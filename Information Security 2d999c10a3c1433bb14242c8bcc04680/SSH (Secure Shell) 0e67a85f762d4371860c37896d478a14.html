SSH (Secure Shell)

SSH (Secure Shell) adalah protokol jaringan yang digunakan untuk mengamankan komunikasi antara dua perangkat. Dengan menggunakan SSH, Anda dapat mengakses dan mengendalikan komputer jarak jauh dengan aman.

SSH (Secure Shell) juga merupakan sebuah protokol jaringan untuk komunikasi data yang aman, layanan remote shell atau perintah eksekusi dan layanan jaringan lainnya yang aman antara dua komputer jaringan yang terhubung melalui saluran aman walaupun melalui jaringan yang tidak aman: server dan klien (masing-masing menjalankan server SSH dan program klien SSH). Spesifikasi protokol membedakan dua versi utama yang disebut sebagai SSH-1 dan SSH-2.

Aplikasi yang paling terkenal dari protokol adalah untuk akses ke account shell pada sistem operasi Unix. Ia dirancang sebagai pengganti Telnet dan lain protokol shell aman seperti rsh Berkeley dan rexec protokol, yang mengirim informasi, terutama password, di plaintext, membuat mereka rentan terhadap intersepsi dan pengungkapan menggunakan analisis paket. Enkripsi yang digunakan oleh SSH dimaksudkan untuk memberikan kerahasiaan dan integritas data melalui jaringan tidak aman, seperti Internet.

Instalasi SSH (Client & Server)

Untuk menginstall SSH di Ubuntu 22.0.4 LTS, Anda dapat mengikuti langkah-langkah berikut:

  1. Buka terminal di Ubuntu
  1. Install openssh-server dan openssh-client
    sudo apt install openssh-server
sudo apt install openssh-client
  1. Setelah instalasi selesai, SSH akan otomatis berjalan di background
    # cek status openssh-server
sudo systemctl status ssh.service

# cek status openssh-client
ssh -V

Konfigurasi SSH

  1. Untuk mengonfigurasi SSH, Anda dapat mengedit file konfigurasi di /etc/ssh/sshd_config menggunakan editor teks seperti Nano atau Vim.
    sudo nano /etc/ssh/sshd_config
  1. Di dalam file konfigurasi, Anda dapat mengatur opsi-opsi seperti port SSH, otentikasi pengguna, dan lain-lain sesuai kebutuhan Anda.
    # Cari value berikut, lalu ubah sesuai yang saya buat
Port 22
PermitRootLogin yes
PubkeyAuthentication yes
    • Port: Opsi ini digunakan untuk mengubah port SSH yang digunakan. Secara default, port SSH adalah 22. Anda dapat mengubahnya menjadi angka port lain untuk meningkatkan keamanan.
    • PermitRootLogin: Opsi ini mengontrol apakah login sebagai pengguna root diizinkan melalui SSH. Jika Anda ingin melarang login sebagai pengguna root, Anda dapat mengaturnya menjadi no.
    • PasswordAuthentication: Opsi ini mengontrol apakah otentikasi dengan kata sandi diizinkan. Jika Anda ingin hanya menggunakan otentikasi dengan kunci publik, Anda dapat mengaturnya menjadi no.
    • PubkeyAuthentication: Opsi ini mengontrol apakah otentikasi dengan kunci publik diizinkan. Jika Anda ingin mengaktifkan otentikasi dengan kunci publik, Anda dapat mengaturnya menjadi yes.
    • AllowUsers: Opsi ini membatasi akses SSH hanya untuk pengguna yang terdaftar. Anda dapat menambahkan daftar nama pengguna yang diizinkan untuk login melalui SSH.
    • AllowTcpForwarding: Opsi ini mengontrol apakah forwarding TCP diizinkan melalui SSH. Forwarding TCP memungkinkan Anda untuk mengirim lalu lintas jaringan melalui koneksi SSH.
    • X11Forwarding: Opsi ini mengontrol apakah X11 forwarding diizinkan melalui SSH. X11 forwarding memungkinkan Anda untuk menjalankan aplikasi GUI dari server jarak jauh.
  1. Setelah selesai mengkonfigurasi, simpan perubahan dan restart layanan SSH dengan menjalankan perintah:
    sudo systemctl restart ssh
  1. Sekarang, SSH sudah terkonfigurasi dan siap digunakan di Ubuntu 22.0.4 LTS.

Uji Coba SSH (Koneksi Antar 2 Device)

Untuk menggunakan SSH, Anda dapat menggunakan perintah ssh di terminal dengan format:

ssh username@alamat-ip-atau-hostname

Misalnya, jika Anda ingin terhubung ke server dengan alamat IP 192.168.100.250 menggunakan nama pengguna "hanz", jalankan perintah:

ssh hanz@192.168.100.250

Kemudian, jika diminta, masukkan kata sandi untuk pengguna tersebut.

Dengan menggunakan SSH, Anda dapat menjalankan perintah-perintah di terminal server jarak jauh dan mengakses file serta layanan yang ada di dalamnya dengan aman.

Automatic Login with RSA/DSA Keys

Kita dapat login ke ssh user@remote_host langsung tanpa menginputkan password setiap login

  1. Bukalah terminal pada laptop client anda, lalu ketik command berikut, untuk generate keys nya
    # Generate key-gen RSA atau DSA terserah anda, pilih salah satu
# Lalu tinggal enter-enter saja, biarkan semuanya default

ssh-keygen -t rsa
ssh-keygen -t dsa

: ' This will generate the keys using the RSA Algorithm. 
At the time of this writing, the generated keys will have 3072 bits. 
You can modify the number of bits by using the -b option. 
For example, to generate keys with 4096 bits, you can do: '

ssh-keygen -t rsa -b 4096 
ssh-keygen -t dsa -b 1024

# Akan muncul file id_rsa dan id_rsa.pub dalam path ~/.ssh/
# Sekarang tinggal kita copy isi dari id_rsa.pub ke ~/.ssh/authorized_keys pada server
  1. Lalu tinggal kita copykan isi dari file id_rsa.pub (kunci publik) ke ~/.ssh/authorized_keys pada device server dengan command seperti berikut
    ssh-copy-id username@alamat-ip-atau-hostname

    Misalnya, jika Anda ingin mengcopy ke server dengan alamat IP 192.168.100.250 menggunakan nama pengguna "hanz", jalankan perintah:

    ssh-copy-id hanz@192.168.100.250

    Nah nanti anda diminta untuk memasukan password device server anda yang ingin anda remote, proses meminta password ini hanya terjadi sekali saja, setelah itu anda akan dapat login bebas dengan cara seperti ini tanpa password lagi.
