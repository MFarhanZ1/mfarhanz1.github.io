Mail Server Localhost CLI Based

Pada contoh kali ini kita akan menggunakan 4 packages buat mengimpelementasikan mail server pada ubuntu, packages tersebut diantara lain yakni: postfix, dovecot-imapd, dovecot-pop3d, dan telnet, nah sekarang mari kita ulas cara kerja antar 4 packages tersebut.

Postfix, dovecot pop3d dan dovecot imapd adalah program yang digunakan untuk membuat server email di Linux. Postfix bertanggung jawab untuk mengirim dan menerima email melalui protokol SMTP (Simple Mail Transfer Protocol). Dovecot pop3d dan dovecot imapd bertanggung jawab untuk menyimpan dan mengambil email dari server melalui protokol POP3 (Post Office Protocol 3) atau IMAP (Internet Message Access Protocol).

Alur kerja dari program-program ini adalah sebagai berikut:

  • Pengirim email menggunakan program seperti Outlook atau Thunderbird sebagai Mail User Agent (MUA) untuk menulis dan mengirim email (bisa menggunakan telnet localhost 25 untuk melakukan ujicoba sementara).
  • MUA menghubungi Postfix sebagai Mail Transfer Agent (MTA) untuk mengirim email ke server tujuan melalui protokol SMTP (Simple Mail Transfer Protocol).
  • Postfix menerima email dan memeriksa apakah alamat penerima valid atau tidak. Jika valid, Postfix menyampaikan email ke Dovecot LDA (Local Delivery Agent) untuk disimpan di server. Jika tidak valid, Postfix mengembalikan email ke pengirim dengan pesan kesalahan.
  • Dovecot LDA menyimpan email di sebuah folder di server sesuai dengan alamat penerima.
  • Penerima email menggunakan program seperti Outlook atau Thunderbird sebagai MUA untuk membaca dan mengunduh email.
  • MUA menghubungi Dovecot pop3d atau dovecot imapd untuk mengambil email dari server melalui protokol POP3 atau IMAP.
  • Dovecot pop3d atau dovecot imapd mengirimkan email yang diminta ke MUA.

Telnet adalah program yang digunakan untuk menguji koneksi dan komunikasi antara program-program di atas. Telnet dapat digunakan untuk mengirim perintah SMTP, POP3 atau IMAP secara manual dan melihat responsnya. Penjelasan lengkap tata cara menggunakan telnet akan di ulas diakhir page ini, oleh karna itu, silahkan disimak baik-baik.

Instalasi Postfix serta Packages Pendukung

# menginstall postfix dan packages pendukungnya
sudo apt install postfix dovecot-imapd dovecot-pop3d -y

Lalu, akan ada pop up permintaan configurasi sementara oleh postfix, begini cara mengisinya:

  1. Nanti akan diberikan penjelasan awal, tinggal pilih OK, lalu tekan enter
  1. Pilih Internet Site karna kita akan uji coba secara local dengan telnet + localhost, lalu pilih Ok
  1. Pada system mail name, isi sembarang domain buat mail server kita, terserah aja, lalu enter aja.

Dan, kelar deh, instalasi postfix beserta packages tambahan dovecot selesai terpasang di device kamu, sekarang kita tinggal melakukan langkah kedua, yakni konfigurasi ulang postfix.

Konfigurasi Postfix

Agar postfix dengan lancar, kita perlu melakukan beberapa konfigurasi, secara interface dpkg postfix maupun secara manual dengan mengedit file confignya:

2.1 Konfigurasi tahap awal secara dpkg-reconfigure postfix

# kita menjalankan command ini buat membuka interface konfigurasi postfix manual
sudo dpkg-reconfigure postfix

Lalu, akan ada pop up permintaan configurasi lebih lengkap oleh postfix, begini cara mengisinya:

  1. Nanti diawal akan sama seperti yang tertera pada tahap 1-3 pada instalasi sebelumnya, kalian tetap biarkan default aja, tinggal enter-enter aja 3 kali, biarkan isinya tetap seperti yang kalian isi saat tahap awal instalasi, jangan di acak-acak.
  1. Nah, lalu di tahap ke 4 baru lah agak berbeda seperti di tahap instalasi, kalian akan menjumpai hal berikut, tinggal kalian kosongkan aja, lalu enter saja.
  1. Lalu, pada tahapan ini kalian diminta masukan nama domain yang dituju di mail server kalian, kalian tinggal biarkan default, lalu kalian tambahkan nama domain yang udah kalian inputkan di awal tadi saat instalasi dalam contoh tadi ialah mail.hanz.com lalu tambahkan juga domain lagi tanpa subdomain mail, dalam contoh ini yakni hanz.com (hapus subdo mail.hanz.com):
  1. Nah, nanti bakal ditanyain “Force Synchronous Update” itu kalian NO Kan saja, lalu enter.
  1. Setelah itu kita akan melakukan specify local network yang bisa di retrieve oleh postfix, biarkan sebelumnya default, lalu kita tambahkan ujungnya 0.0.0.0/0 lihat contoh dibawah ini:
  1. Setelah itu tinggal kalian OK kan aja pada mail box size limit (default: 0), sama local address extension character (default: +) biarkan default semua nya, seperti dibawah ini:

Selesai deh, postfix selesai di konfigurasi untuk tahap awal, kita akan masuk ke tahap yang kedua.

2.2 Konfigurasi tahap kedua secara manual dengan nano

Pada tahap ini, kita akan melakukan editing 4 buah file config, yakni main.cf, dovecot.conf, 10-auth.conf, dan terakhir 10-mail.conf, kita mulai satu persatu pengubahannya:

  1. Mengedit config main.cf
    # membuka teks editor nano buat mengedit config di main.cf
sudo nano /etc/postfix/main.cf

    Pergi ke bagian paling bawah baris, lalu kalian edit isi 2 hal dibawah ini:

    myhostname = mail.hanz.com (isi sesuai nama domain yang kalian buat saat instalasi) home_mailbox = Maildir/
    Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, setelah itu kalian tinggal teken ctrl + x buat keluar dari nano text editor
  1. Mengedit config dovecot.conf
    # membuka teks editor nano buat mengedit config di dovecot.conf
sudo nano /etc/dovecot/dovecot.conf

    tekan ctrl + w, lalu kalian cari “listen” terus itu enter, nah kalau dah ketemu kalian ubah kek gini:

    listen = *
    Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, setelah itu kalian tinggal teken ctrl + x buat keluar dari nano text editor
  1. Mengedit config 10-auth.conf
    # membuka teks editor nano buat mengedit config di 10-auth.conf
sudo nano /etc/dovecot/conf.d/10-auth.conf

    Kalian nanti akan meliat disable_plaintext_auth, nah kalau dah nampak kalian ubah kek gini:

    disable_plaintext_auth = no
    Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, setelah itu kalian tinggal teken ctrl + x buat keluar dari nano text editor
  1. Mengedit config 10-mail.conf
    # membuka teks editor nano buat mengedit config di 10-mail.conf
sudo nano /etc/dovecot/conf.d/10-mail.conf

    Tekan ctrl + w, lalu find “mail_location", nah kalau dah nampak kalian ubah kek gini:

    remove sign (#): mail_location = maildir:~/Maildir add sign (#): mail_location = mbox:~/mail:INBOX=/var/mail/%u
    Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, setelah itu kalian tinggal teken ctrl + x buat keluar dari nano text editor

Kelar deh semua, terakhir kita akan membuat perintah lanjutan terakhir untuk membuat direktori penyimpanan Maildir/, command nya yakni seperti dibawah ini:

# untuk membuat direktori penyimpanan Maildir/
sudo maildirmake.dovecot /etc/skel/Maildir

Selesai deh proses konfigurasi postfix dan dovecot nya, sekarang kalian hanya perlu merestart service dovecot dan postfix kalian dengan 2 command berikut ini, jalankan berurutan:

# restart kedua buah service dibawah ini dengan command berikut
sudo service postfix restart
sudo service dovecot restart

Sekarang kita akan melakukan tahap uji coba secara local dengan telnet localhost 25 dan 110.

Uji coba Postfix secara local via Telnet

Pertama kita tambahkan dulu dua buah user baru yang akan bertindak sebagai pengirim dan penerima email nantinya, berikut caranya:

# cara membuat user baru, isi nama_user sesuai nama yang kalian mau
sudo adduser <nama_user>
sudo adduser <nama_user>

# misal kita akan membuat user romeo sebagai pengirim, dan juliet sebagai penerima

sudo adduser juliet

3.1 Cara mengirim email ke user lain

# konek ke server lokal dulu via telnet (port 25 untuk sending email)
telnet localhost 25

Seteleh terkoneksi, silahkan isi hal berikut pada telnet console nya, enter saat tulis tiap baris:

'
	Tuliskan satu persatu ya pada menu console telnetnya!
	1. Setelah menulis tiap baris, tinggal tekan enter
	2. Nanti telnet akan memberikan respons tiap kita enter
	3. Jika respons OK berarti command anda betul, jika eror, berarti typo tuh'

# tata cara penulisan pada console telnet localhost 25
mail from: romeo
rcpt to: juliet
data
<your message here>
.
quit

3.2 Cara menerima email dari user lain

# konek ke server lokal dulu via telnet (port 110 untuk receive email)
telnet localhost 110

Seteleh terkoneksi, silahkan isi hal berikut pada telnet console nya, enter saat tulis tiap baris:

'
	Tuliskan satu persatu ya pada menu console telnetnya!
	1. Setelah menulis tiap baris, tinggal tekan enter
	2. Nanti telnet akan memberikan respons tiap kita enter
	3. Jika respons OK berarti command anda betul, jika eror, berarti typo tuh'

# tata cara penulisan pada console telnet localhost 110
user juliet
pass 123321
stat # bakal kasih notif ada berapa pesan yang masuk
rcpt 1 # ganti angka 1 dengan angka pesan nomor berapa yang mau dibaca
quit
