Mail Server Webmail Based

SquirrelMail adalah salah satu aplikasi webmail yang populer yang dirancang untuk dijalankan di server web. Ini memungkinkan pengguna untuk mengakses kotak surat email mereka melalui antarmuka web yang sederhana dan mudah digunakan. SquirrelMail sering kali digunakan sebagai solusi webmail di server Linux, termasuk Ubuntu.

Berikut adalah beberapa poin tentang SquirrelMail:

  1. Antarmuka Pengguna Web: SquirrelMail menyediakan antarmuka web yang bersahabat dan mudah digunakan, memungkinkan pengguna untuk membaca, mengirim, dan mengelola email mereka melalui browser web.
  1. Ringan dan Mudah Dipasang: Aplikasi ini terkenal karena kesederhanaannya dan persyaratan sistem yang ringan. Hal ini memungkinkan SquirrelMail diinstal dan berjalan di server dengan sumber daya yang terbatas.
  1. Kemampuan Kustomisasi: Meskipun sederhana, SquirrelMail memiliki beberapa fitur kustomisasi yang memungkinkan administrator untuk menyesuaikan tampilan dan fungsionalitasnya sesuai kebutuhan.
  1. Dukungan Protokol: SquirrelMail mendukung protokol email umum seperti POP3 dan IMAP, memungkinkan pengguna untuk mengakses email dari server yang mendukung protokol tersebut.
  1. Open Source: Sebagai perangkat lunak open-source, SquirrelMail memiliki kode sumber yang terbuka, yang memungkinkan pengembang untuk memodifikasinya dan mengadaptasinya sesuai kebutuhan mereka.
  1. Tidak Berkembang Lagi: Perlu dicatat bahwa SquirrelMail sekarang tidak lagi mendapatkan pengembangan aktif. Meskipun masih banyak digunakan, beberapa organisasi lebih memilih solusi webmail yang lebih baru dan didukung secara aktif seperti Roundcube atau RainLoop.

Sekedar informasi, SquirrelMail sudah tak lagi tersedia di ubuntu apt package manager, oleh karna itu untuk megunduh dan menginstallnya kita memerlukan cara lain, yakni menggunakan wget, untuk ulasan lebih lengkapnya akan di ulas pada sub-bab instalasi squirrelmail.

Instalasi Squirrel-Mail

Pastikan sebelum masuk ke instalasi Squirrel mail anda wajib telah mengikuti semua langkah yang terdapat pada instalasi webmail localhost cli based, jika belum silahkan ikuti dulu panduan instalasi dan konfigurasi semua packages nya, barulah anda bisa melanjutkan ke tahap squirrel-mail ini bro.

Okey, kita masuk ke pembahasan Squirrel-Mail, sekedar informasi nih, Squirrel-Mail sudah tak tersedia lagi di ubuntu versi 16.04 keatas, untuk menyiasatinya, kita dapat mengunduhnya dengan perintah wget serta unzip untuk mengekstraknya:

# menginstall wget dan unzip sebagai tools penginstall squirrel-mail
sudo apt install wget
sudo apt install unzip

# mengunduh squirrelmail melalui wget (squirrelmail sudah tak tersedia di apt)
cd ~/Downloads/ && sudo wget https://sourceforge.net/projects/squirrelmail/files/stable/1.4.22/squirrelmail-webmail-1.4.22.zip

# mengekstrak squirrelmail yang telah di download tadi
# nanti akan muncul folder 'squirrelmail-webmail-1.4.22/' disinilah hasil ekstrak tadi
unzip squirrelmail-webmail-1.4.22.zip 

# lalu kita buat folder di folder apache2 untuk menyimpan data dari squirrelmail
sudo mkdir /var/www/mail

# kita copy kan hasil ekstrak tadi ke folder dalam apache2 yang udah kita buat tadi
sudo cp -r squirrelmail-webmail-1.4.22/ /var/www/mail/squirrelmail

# lalu kita ubah folder ownershipnya, dan add permission nya
sudo chown -R www-data:www-data /var/www/mail/squirrelmail/
sudo chmod 755 -R /var/www/mail/squirrelmail/

Kelar deh instalasi manual dari squirrelmail, sekarang kita akan masuk ke tahap selanjutnya, yakni tahap konfigurasi squirrelmail via apache, simak lanjutannya dibawah ini.

Konfigurasi Squirrel-Mail

Agar Squirrel-Mail dapat berjalan kita harus melakukan beberapa tahapan konfigurasi, diantaranya, pertama kita harus mengedit /etc/hosts untuk menambahkan penamaan dns jaringan baru. 

# membuka nano text editor untuk mengedit file hosts
sudo nano /etc/hosts

pastikan configurasi kalian sesuai dengan gambar dibawah ini, ubah domain sesuai domain name yang kalian inputkan pada saat konfigurasi postfix tahap awal, lihat baik-baik ss dibawah ini:

Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, setelah itu kalian tinggal teken ctrl + x buat keluar dari nano text editor

Lalu kita akan melakukan penambahan config baru didalam folder /etc/apache2/sites-available kita harus mengcopykan file 000-default.conf ke dalam folder yang sama menjadi nama file baru yakni mail.conf berikut one-line command singkatnya:

# penambahan config baru dengan cara mengcopykan template file default.conf menjadi nama file config baru
cd /etc/apache2/sites-available && sudo cp 000-default.conf mail.conf

nah ges, barulah muncul mail.conf itulah yang nanti akan kita edit isinya, pertama kita akan buka terlebih dahulu nano text editor untuk memperbarui isi file mail.conf tersebut, berikut caranya:

# membuka nano text editor buat mengedit isi file mail.conf
sudo nano mail.conf

Hal yang akan kita ubah disini yakni hanya 2 saja, yaitu ServerName dengan DocumentRoot, lihat gambar dibawah, pastikan sesuai, lihat penjelasan dibawah gambar agar tidak misskom kalian:

Nah, setelah itu kita akan memberitahu kepada apache2 bahwa config mail.conf telah aktif sekarang, tujuannya agar bisa di akses nama domain virtual tersebut. Kita juga harus mematikan config default dari apache2 agak tidak bentrok berjalan keduanya, dan diakhir kita akan merestart service dari apache2, simak langkah-langkahnya dibawah ini yak ges:

# hidupkan mail.conf dengan a2ensite, dan matikan default conf dengan a2ensite conf
sudo a2ensite mail.conf
sudo a2dissite 000-default.conf

# reload dan restart service dari apache2 agar perubahan terbaca
sudo systemctl reload apache2
sudo systemctl restart apache2

# membuka file konfigurasi bawaan dari squirrelmail
sudo perl /var/www/mail/squirrelmail/config/conf.pl

2.1 Konfigurasi Tambahan Khusus Buat Squirrel-Mail di Ubuntu 16.04+

Sekedar informasi, SquirrelMail akan blank jika langsung kita jalankan, karna squirrel mail sudah tak lagi dikembangkan untuk versi ubuntu 16.04 keatas, dan sudah tak bisa lagi dijalankan pada apache2 dengan versi php7.0 keatas, oleh karna itu, kita perlu melakukan downgrade versi php7.0 keatas menjadi versi php 5.6 khusus buat apache2 nya, berikut caranya:

# add the Ondrej PHP repository to your Ubuntu, This repository contains all the released versions of PHP to date.
sudo add-apt-repository ppa:ondrej/php

# avoid locale repo adding bug error
sudo locale-gen id_ID.UTF-8

# install php version 5.6 to your ubuntu
sudo apt install php5.6

# matikan service php8.1 buat apache, dan hidupkan yang php versi 5.6
sudo a2dismod php8.1 # cek dulu versi php kita yang akan dimatikan dengan command php -v
sudo a2enmod php5.6

# restart semua service yang berkaitan dengan squirrelmail
sudo systemctl restart apache2
sudo service dovecot restart
sudo /etc/init.d/apache2 restart
sudo /etc/init.d/postfix restart
