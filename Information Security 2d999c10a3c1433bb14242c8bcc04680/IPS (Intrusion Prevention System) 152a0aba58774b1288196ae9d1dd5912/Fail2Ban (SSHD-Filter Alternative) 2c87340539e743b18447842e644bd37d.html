Fail2Ban (SSHD-Filter Alternative)

Fail2Ban adalah perangkat lunak keamanan yang digunakan untuk melindungi server dari serangan brute-force dan serangan otentikasi yang tidak sah. Salah satu fungsinya yang paling umum adalah memantau log keamanan pada server dan secara otomatis memblokir alamat IP yang mencoba masuk secara berulang-ulang dengan informasi otentikasi yang salah.

Serangan brute-force yang dijaga oleh Fail2Ban biasanya terjadi terhadap layanan yang memerlukan otentikasi seperti SSH (Secure Shell), FTP (File Transfer Protocol), HTTP (Hypertext Transfer Protocol), dan layanan lain yang dapat diakses dari jaringan yang terhubung ke internet.

Fail2Ban bekerja dengan memantau log keamanan dari layanan-layanan ini untuk mendeteksi pola serangan brute-force, di mana ada percobaan otentikasi yang gagal secara berulang dari alamat IP yang sama dalam jangka waktu tertentu. Ketika pola seperti itu terdeteksi, Fail2Ban secara otomatis akan memblokir atau membatasi akses dari alamat IP yang bersangkutan.

Instalasi Fail2Ban

It operates by monitoring log files for certain types of entries and runs predetermined actions based on its findings. You can install the software with the following

sudo apt install fail2ban

Once installed, copy the default jail.conf file to make a local configuration with this command

sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local

Konfigurasi Fail2Ban

Untuk dapat melakukan hal-hal tertentu diluar dari instalasi default, kita bisa melakukan konfigurasi manual dengan mengedit file config /etc/fail2ban/jail.local dengan command berikut:

# Edit file konfigurasinya dengan command
sudo nano /etc/fail2ban/jail.local

nah saat masuk ke dalam text editor file jail.local silahkan cari baris berikut, dan anda dapat mengubahnya sesuka hati anda, sesuai kebutuhan:

[DEFAULT]
ignoreip = 127.0.0.1
bantime  = 1m
findtime = 5m
maxretry = 2

Ini adalah contoh konfigurasi dari file fail2ban.conf atau jail.conf. Konfigurasi ini mengatur beberapa parameter utama untuk pengaturan Fail2Ban, berikut keterangan masing fungsinya:

  1. ignoreip: Menentukan alamat IP yang akan diabaikan oleh Fail2Ban. Dalam contoh ini, 127.0.0.1 (localhost) diabaikan, yang berarti percobaan otentikasi yang gagal dari alamat IP ini tidak akan mengakibatkan blokir oleh Fail2Ban.
  1. bantime: Merupakan periode waktu dalam detik di mana alamat IP yang diblokir akan tetap diblokir oleh Fail2Ban sebelum dihapus dari daftar blokir. Dalam contoh ini, bantime disetel ke 1m (1 menit), kita juga bisa setel dalam satuan detik seperti 3600 detik atau 1 jam.
  1. findtime: Merupakan periode waktu dalam detik yang menjadi timer waktu untuk mencari percobaan otentikasi yang gagal. Jika jumlah percobaan gagal terjadi dalam jangka waktu ini, maka Fail2Ban akan mengambil tindakan. Dalam contoh ini, findtime disetel ke 5m (5 menit) kita juga dapat setel menjadi 600 detik atau 10 menit.
  1. maxretry: Jumlah maksimum percobaan otentikasi yang diizinkan dalam periode findtime sebelum alamat IP diblokir. Dalam contoh ini, maxretry disetel ke 3, yang berarti jika terdapat 3 percobaan otentikasi yang gagal dalam 10 menit, alamat IP tersebut akan diblokir selama 1 jam.

Konfigurasi ini membantu untuk mengurangi risiko serangan brute-force dengan membatasi jumlah percobaan otentikasi yang gagal dari alamat IP tertentu. Pastikan untuk menyesuaikan konfigurasi ini dengan kebutuhan keamanan dan karakteristik lingkungan jaringan Anda.

Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, setelah itu kalian tinggal teken ctrl + x buat keluar dari nano text editor
And all done, tinggal restart service dengan $ sudo systemctl restart fail2ban selesai deh brow, fail2ban udah bisa digunakan

Uji Coba Bruteforce Password SSH Login

Silahkan buka terminal pada laptop client, dan ketik ini, lalu ketika input password, masukan ngasal:

ssh [username]@[hostname/ip_address]

Terus masukan password ngasal, jika salah passwordnya udah melebihi batas yang di tetapkan di variabel maxretry pada config jail.local sebelumnya, nanti laptop client mendadak akan nge-lag saat mau login ssh ke server nya, nah itu artinya ip client/ip kita udah di blokir sama fail2ban client, dan akan otomatis ter-unblock sesuai bantime yang kita tetapkan tadi, dalam contoh ini saya menetapkan 1m (1 menit).

3.1 Melakukan Ban/UnBan Fail2Ban IP Jail via Manual

Sebenarnya fail2ban akan secara otomatis melakukan unban setelah masa bantime selesai (dalam config sebelumnya selama 1 menit), namun jika kalian ingin membuka ban tersebut sebelum masa ban nya selesai, bisa kok, caranya gini, pertama silahkan buka terminal pada laptop server yang berjalan service fail2ban-nya, dan ketik ini:

# Template command with args
sudo fail2ban-client set <jail> banip/unbanip <ip address>

# For example: unblocking ip 192.168.100.250
sudo fail2ban-client set sshd unbanip 192.168.100.250

kelar deh fail2ban-client secara manual akan menjalan request anda, baik itu ban ataupun unban, sangat simpel mengoperasikan fail2ban, tidak memerlukan effort yang lebih.

Monitoring Log Fail2Ban Secara Live

Kalian bisa memonitoring status apa saja yang sedang terjadi di service fail2ban, seperti: status ada ip yang sedang mencoba bruteforce server ssh kita, atau status fail2ban sedang melakukan banned/unbanned suatu ip yang sedang mencoba bruteforce ke ssh kita, caranya mudah, silahkan buka terminal pada laptop server, lalu ketik command seperti berikut:

sudo tail -f /var/log/fail2ban.log

# jika ingin tampilan dengan UI lebih menarik, silahkan ketikan ini:
sudo lnav /var/log/fail2ban.log

semua status log yang sedang terjadi di service fail2ban akan keluar secara live di terminal anda, anda bisa memonitor status ip yang sedang di banned oleh fail2ban dan sebaliknya.
