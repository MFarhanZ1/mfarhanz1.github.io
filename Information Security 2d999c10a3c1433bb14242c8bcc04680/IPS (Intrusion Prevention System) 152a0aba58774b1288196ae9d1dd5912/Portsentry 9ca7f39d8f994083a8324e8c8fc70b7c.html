Portsentry

Portsentry adalah alat keamanan pada sistem Linux yang berfungsi untuk mendeteksi dan melindungi dari serangan pemindaian port yang tidak sah atau upaya penetrasi (penetration attempts). Alat ini bekerja dengan cara memonitor aktivitas port pada sistem dan merespons terhadap aktivitas yang mencurigakan dengan memblokir atau melaporkannya.

Portsentry dapat diintegrasikan dengan berbagai perangkat lunak keamanan lainnya, seperti firewall, untuk meningkatkan lapisan perlindungan sistem. Alat ini memiliki kemampuan untuk merespons terhadap serangan dari berbagai jenis pemindaian port, termasuk serangan SYN, NULL, FIN, XMAS, dan pemindaian port UDP.

Dengan kemampuan deteksinya, Portsentry membantu mengurangi potensi kerentanan sistem terhadap serangan port dan memberikan waktu bagi administrator untuk mengambil tindakan pencegahan atau memperbaiki kerentanan sebelum serangan yang lebih serius terjadi.

Instalasi Portsentry

Dalam sistem operasi Ubuntu, Portsentry sudah tersedia di repository dan bisa diinstall menggunakan paket manager apt. Caranya dengan menjalankan perintah berikut

sudo apt install portsentry

Tunggu proses instalasi portsentry beserta dependensinya selesai. Namun perlu diketahui bahwa secara default Portsentry hanya mendeteksi adanya scanning, tapi tidak memblokirnya. Nah, jika kalian ingin memblokir serangan tersebut, kalian bisa edit file confignya.

Konfigurasi Portsentry

Portsentry berjalan di beberapa mode yang berbeda, diantaranya ada -tcp, -udp, -atcp, etc, nah singkatnya -atcp adalah mode tertinggi dari portsentry yang bisa menghalau hampir semua serangan port scanning, baik itu connect scans, half-open scans, FIN scans, NULL scans, XMAS scans dan UDP scans, nah sekarang kita akan mengkonfigurasi portsentry untuk berjalan di mode tertinggi yakni mode -atcp, berikut cara konfigurasinya, silahkan disimak baik-baik:

# Kita akan mengedit default porsentry args mode saat service di start
sudo nano /etc/default/portsentry

lalu, ubah sesuai ini, TCP_MODE=”atcp” dan UDP_MODE=”sudp”

jika kalian dibiarkan default tcp_mode=”tcp” dan udp_mode=”udp", maka portsentry hanya akan bisa mendetect -sT dan -sU scan saja, lalu jenis scan -sS -sF -sU -sX malah akan dihiraukan oleh portsentry.

wajib sesuai diatas ya!

Uji Coba Portsentry (NMAP Port Scanning)

Sedikit hal yang harus anda ingat, Portsentry dan Firewall (UFW) berjalan bersamaan dan saling beriringan, oleh karna itu, untuk mendapatkan hasil yang akurat saat ingin menguji coba portsentry, anda harus menonaktifkan sementara Firewall (UFW) dengan command sudo ufw disable kelar.

  • TCP Connect Scan (-sT)
    # melakukan TCP Connect Scan pada server dengan IP 192.168.100.250
nmap -T4 -sT 192.168.100.250
  • TCP SYN (Stealth) Scan (-sS)
    • Fungsi: Mengirim permintaan koneksi SYN tetapi tidak menyelesaikannya.
    • Cara Kerja: Mengirim permintaan koneksi awal ke port target tanpa menyelesaikan tiga langkah koneksi TCP. Nmap sends a SYN packet to the target and waits for a response. If the target responds with a SYN/ACK packet, the port is considered open and ready to establish a connection.
    • Tujuan: Mendeteksi apakah port tertentu terbuka atau tertutup berdasarkan respons dari permintaan koneksi SYN.
    # melakukan TCP SYN (Stealth) Scan pada server dengan IP 192.168.100.250
nmap -sS 192.168.100.250
  • FIN Scan (-sF)

    Scan ini mengirim paket FIN ke target. Jika target tidak merespon, maka port tersebut dianggap terbuka, begitu sebaliknya, cara melakukannya yakni:

    # melakukan FIN Scan pada server dengan IP 192.168.100.250
nmap -sF 192.168.100.250
  • NULL Scan (-sN)

    Scan ini mengirim paket tanpa flag apapun yang mana akan membuat target menjadi kebingungan, hingga target tidak akan merespon jika port terbuka, jika target malah mengirimkan packet RST berarti port tersebut tertutup, begini cara scannya:

    # melakukan NULL Scan pada server dengan IP 192.168.100.250
nmap -sN 192.168.100.250
  • XMAS Scan (-sX)

    Scan ini mengirim paket dengan flag FIN, URG, dan PSH, mirip dengan lampu pohon Natal (Xmas). Jika target tidak merespon, maka port tersebut dianggap terbuka, begini caranya:

    # melakukan XMAS Scan pada server dengan IP 192.168.100.250
nmap -sX 192.168.100.250
  • UDP Connect Scan (-sU)

    Scan ini mengirim paket UDP ke target. Jika target merespon dengan pesan ICMP Port Unreachable, maka port tersebut dianggap tertutup, begini cara scanning-nya:

    # melakukan UDP Connect Scan pada server dengan IP 192.168.100.250
nmap -sU 192.168.100.250

3.1 Melakukan Unblock Porsentry IP via Hosts Deny Manual

Portsentry tidak akan melakukan unblock ip secara otomatis, kita harus melakukan unblocking secara manual agar tetap bisa melakukan uji coba portsentry dengan teknik nmap scanning lainnya.

# Step 1: Cek dulu list ip yang telah di blokir (ditandain kenak flag reject deny !H)
route -n
nah, IP yang terdapat flags !H itu berarti terkena blokir oleh portsentry.

Sekarang kita masuk ke contoh kasus, semisalkan nih berdasarkan data route diatas kita pengen melakukan unblock terhadap IP 192.168.100.53, anda cukup mengetik command ini:

# Template contoh command buat unblocking IP di dalem route
sudo route del -host <IP> reject

# Case Example: Coba Unblocking IP 192.168.100.53 from route
sudo route del -host 192.168.100.53 reject

Terus, anda belum selesai, sekarang edit file hosts.deny berikan tanda pagar (#) dibagian ip terkait, untuk melakukan hal tersebut, anda bisa mengetik command seperti berikut:

# edit list ip yang dihiraukan portsentry
sudo nano /etc/hosts.deny

nah, kayak gambar disebelah, tinggal kamu kasih aja tanda pagar (#) buat IP yang mau kamu unblock (IP nya samain dengan yang udah di delete di route tadi)

Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, setelah itu kalian tinggal teken ctrl + x buat keluar, kelar deh, nmap scanning dari client udah bisa kembali dilakukan.
And all done, tinggal restart service dengan $ sudo systemctl restart portsentryselesai deh brow, portsentry udah bisa digunakan kembali untuk memblokir ip tersebut

Monitoring Log Portsentry Secara Live

Kalian bisa memonitoring status apa saja yang sedang terjadi di service portsentry, seperti: status ada ip yang sedang mencoba port scanning server kita (attackalert), atau status portsentry sedang melakukan banned/unbanned suatu ip yang sedang mencoba port scanning nmap ke server kita, caranya mudah, silahkan buka terminal pada laptop server, lalu ketik command seperti berikut:

# Melihat log secara global (semua log proses keseluruhan os ubuntu)
sudo tail -f /var/log/syslog

# Melihat log ip yang di block portsentry
sudo tail -f /var/lib/portsentry/portsentry.history

# Atau cara lebih UI Advance Monitoring, bisa gunakan
sudo lnav /var/lib/portsentry/portsentry.history

semua status log yang sedang terjadi di service portsentry akan keluar secara live di terminal anda, anda bisa memonitor status ip yang sedang di banned oleh portsentry dan sebaliknya.
