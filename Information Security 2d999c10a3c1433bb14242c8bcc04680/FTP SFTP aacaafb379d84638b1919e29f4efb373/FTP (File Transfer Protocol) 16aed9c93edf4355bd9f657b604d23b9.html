FTP (File Transfer Protocol)

File Transfer Protocol (FTP) adalah protokol TCP untuk mengunduh file antar komputer. Di masa lalu, ini juga digunakan untuk mengunggah tetapi, karena metode tersebut tidak menggunakan enkripsi, kredensial pengguna serta data yang ditransfer jelas dan mudah disadap. Jadi jika Anda mencari cara untuk mengunggah dan mengunduh file dengan aman, lihat dokumentasi OpenSSH .

FTP bekerja pada model klien/server. Komponen server disebut daemon FTP . Itu terus mendengarkan permintaan FTP dari klien jarak jauh. Ketika permintaan diterima, ia mengelola login dan mengatur koneksi. Selama sesi berlangsung, ia menjalankan perintah apa pun yang dikirim oleh klien FTP.

Instalasi FTP Server (File Transfer Protocol) dengan [vsftpd]

vsftpd is an FTP daemon available in Ubuntu. It is easy to install, set up, and maintain. To install vsftpd you can run the following command:

sudo apt install vsftpd

Konfigurasi FTP Server

Untuk dapat melakukan hal-hal tertentu diluar dari instalasi default, kita bisa melakukan konfigurasi manual dengan mengedit file config /etc/vsftpd.conf dengan command berikut:

# Izinkan firewall terlebih dahulu dengan command
sudo ufw allow from any to any port 20,21,22,990,40000:50000 proto tcp

# Edit file konfigurasinya dengan command
sudo nano /etc/vsftpd.conf

nah saat masuk ke dalam text editor file vsftpd.conf pergi ke paling bawah, lalu tambahkan beberapa baris berikut, sesuai list dibawah ini:

# Hilangkan comment pada perintah dibawah
write_enable=YES
chroot_local_user=YES

# Tambahkan ini dibawah chroot_local_user=YES
allow_writeable_chroot=YES

pasv_enable=YES
pasv_min_port=40000
pasv_max_port=50000

Jika selesai, teken ctrl + o lalu enter buat menyimpan perubahan, Lalu teken ctrl + x buat keluar

2.1 Konfigurasi Tambahan SSL/TLS FTP Server (Optional)

SSL (Secure Sockets Layer) dan TLS (Transport Layer Security) adalah protokol keamanan yang digunakan untuk melindungi komunikasi antara klien dan server dengan menyediakan enkripsi dan keotentikasian data yang ditransmisikan. Dalam konteks FTP (File Transfer Protocol), SSL dan TLS dapat digunakan untuk membuat koneksi FTP menjadi lebih aman, dikenal sebagai FTPS (FTP Secure), berikut cara memasang SSL/TLS pada FTP Server:

# Generate certificate RSA 4096 bit
sudo openssl req -x509 -nodes -days 365 -newkey rsa:4096 -keyout /etc/ssl/private/vsftpd.key -out /etc/ssl/certs/vsftpd.pem

Untuk apply SSL/TLS pada FTP Server, kita bisa melakukan konfigurasi manual dengan mengedit file config /etc/vsftpd.conf dengan menambahkan pada paling bawah file baris seperti berikut:

ssl_enable=YES
allow_anon_ssl=NO

# FIX 530 Non-anonymous sessions must use encryption. (CLI ftp can't login with ssl/tls)
force_local_data_ssl=NO
force_local_logins_ssl=NO

ssl_tlsv1=YES
ssl_sslv2=NO
ssl_sslv3=NO
ssl_ciphers=HIGH

rsa_cert_file=/etc/ssl/certs/vsftpd.pem
rsa_private_key_file=/etc/ssl/private/vsftpd.key

And all done, tinggal restart service dengan $ sudo systemctl restart vsftpd selesai deh brow.

Uji Coba Koneksi ke-FTP Server

Untuk dapat terkoneksi ke FTP Server, silahkan buka terminal pada laptop client, dan ketik ini:

ftp [username]@[hostname/ip_address]

Lalu masukan password, dan enter, selesai deh, anda telah terhubung ke FTP Server. Jika ingin pake GUI bisa dari bawaan Ubuntu, atau ga pake FileZilla, tinggal konek seperti biasa aja, nanti bakal ada notifikasi SSL/TLS Certificate, tinggal di oke kan, kelar deh.

# Setelah terkoneksi, untuk melihat perintah lengkap bisa tulis "?" sama dengan help
ftp> ?

# Untuk menjalankan command local bisa menggunakan tanda "!" di depan command
ftp> !cd 
ftp> !ls

# Untuk menjalankan command pada device remote, seperti biasa aja
ftp> cd
ftp> ls

# Untuk mendownload file remote ke local
ftp> get <nama_file>
ftp> get test.txt

# Untuk mengupload file local ke remote
ftp> put <nama_file>
ftp> put test.txt
